## 주요 기능

### 1. 자동 변경 감지 & 업로드

- **10분마다** 노션 페이지의 수정 여부(`last_edited_time`)를 확인합니다.
- 변경이 감지되면 **원문 그대로 즉시** 1000.school에 업로드합니다.
- 내용이 비어있으면 업로드를 건너뜁니다.

---

### 2. Gemini AI 자동 다듬기 (Polish)

작성이 끝났다고 판단되면 AI가 내용을 정형화해 최종본으로 덮어씁니다.

**Polish 실행 여부 결정 흐름**

```
매 10분 루프
    │
    ▼
오늘 polish가 이미 완료됐나?
    │ YES → 스킵 (하루 1번만)
    │ NO
    ▼
마지막 변경 후 30분 이상 지났나?
    │ NO → 아직 작성 중, 대기
    │ YES
    ▼
Tomorrow(내일 우선순위) 섹션에 내용이 있나?
    │ YES → ✨ Gemini Polish 실행
    │ NO
    ▼
지금이 오전 8:50~8:59 사이인가?
    │ YES → ⏰ 강제 Polish 실행 (9시 데드라인 전 폴백)
    │ NO → 계속 대기
```

> **폴백 이유**: Tomorrow를 못 적어도 9시가 되면 날짜가 넘어가기 때문에,
> 8:50~8:59 사이에 내용이 있으면 강제로 다듬어 제출합니다.

**Gemini API 실패 시 동작**

```
Gemini API 호출 (최대 3회 재시도)
    │ 성공 → 다듬어진 최종본 업로드
    │ 실패 (quota 초과 등) → ⚠️ 원문 그대로 업로드 (중단 없음)
```

### 3. 역방향 동기화 (1000.school → 노션)

1000.school에 피드백이 달리거나 외부에서 수정된 내용을 노션에 자동 반영합니다.

**단순 덮어쓰기의 문제점**
```
매 10분마다 무조건 덮어쓰면:
  노션에서 작성 중 → 내용이 갑자기 사라졌다 나타남 (깜빡임 💀)
```

**해시 비교로 해결**
```
업로드 성공
    │
    ▼
업로드한 내용을 해시값으로 변환해 저장
예) "오늘 회의..." → MD5 → "a3f9b2c1..."

    ▼ (10분마다)

1000.school 현재 내용 해시 계산
    │
    ├── 저장된 해시와 같다 → 스킵 ✅ (깜빡임 없음)
    └── 저장된 해시와 다르다 → 노션 업데이트 ✅ (피드백 반영)
```

> **해시(Hash)**: 텍스트를 고정 길이의 지문으로 변환. 내용이 1글자라도 다르면 완전히 다른 값이 나옴.
> 전체 텍스트를 비교하지 않아도 변경 여부를 즉시 판단할 수 있음.

### 5. AI 감독 리포트

전체 스니펫 흐름을 분석해 노션 리포트 페이지에 작성합니다.

**7개 분석 지표**

| 지표 | 무엇을 보나 | 높으면 |
|------|------|------|
| 🔥 번아웃 위험도 | 헬스 점수 저하, 피로 패턴 반복 | 위험 ⚠️ |
| 👥 팀 건강도 | 팀 기여 빈도, 갈등 조짐 | 좋음 ✅ |
| 💪 성실도 | 작성 연속성, 내일 목표 달성률 | 좋음 ✅ |
| 🔁 문제 재발성 | 동일 로우라이트 반복 여부 | 재발 많음 ⚠️ |
| 🧠 성장 지수 | 새 학습, 피드백 이행 여부 | 좋음 ✅ |
| ⚡ 실행 집중도 | 목표와 실제 행동 일치도 | 좋음 ✅ |
| 💬 감정 에너지 | 글의 긍정도, 자기효능감 변화 | 긍정적 ✅ |

**토큰 절약 구조**

```
날짜별 스니펫 전체 원문 (~1,000자/일 × N일)
    │
    │  직접 보내면? → 토큰 폭발 💥
    │
    ▼  DB에서 핵심 필드만 추출
날짜 | 헬스 | 하이라이트 요약 | 로우라이트 요약 | 내일 목표
2026-03-19 | 6/10 | API 연동 성공 | 재시험 아쉬움 | API 마무리
2026-03-18 | 5/10 | ...           | ...           | ...
    │
    ▼  압축 요약본 (~100자/일) → Gemini 전송
7개 지표 분석 결과 → 노션 리포트 페이지
```

## 노션 데일리 템플릿

매일 오전 9시에 생성되는 기본 양식입니다.

```
## What              ← 오늘 한 일
## Why               ← 수행 목적
## Team Value Added  ← 팀 기여
## Highlight         ← 잘된 점
## Lowlight          ← 아쉬운 점
## Tomorrow          ← 내일 우선순위  ★ Polish 트리거 역할
```

> `Tomorrow` 섹션에 내용이 채워지면 "오늘 작성 완료"로 판단해 Polish를 준비합니다.

`.env.example` 을 복사해서 `.env` 로 저장하고 값을 채웁니다.

```bash
cp .env.example .env
```

```env
NOTION_TOKEN=secret_xxxx       # 노션 Integration 토큰
NOTION_PAGE_ID=xxxx            # 데일리 페이지 부모 페이지 ID
SCHOOL_API_KEY=I_wrdo_xxxx     # 1000.school API 키
GEMINI_API_KEY=AIzaSy_xxxx     # Gemini API 키
```

**노션 NOTION_PAGE_ID 찾는 방법**
```
노션 부모 페이지 열기
→ 주소창: notion.so/abc123def456ghi789...
→ 마지막 32자리가 PAGE_ID (하이픈 없이 입력)
```

**노션 Integration 연결 방법**
1. [notion.so/my-integrations](https://www.notion.so/my-integrations) → Integration 생성
2. 부모 페이지 우측 상단 `···` → **연결** → 생성한 Integration 선택

**Gemini API 키 발급** (무료)
- [aistudio.google.com/app/apikey](https://aistudio.google.com/app/apikey)
